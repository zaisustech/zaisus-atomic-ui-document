ESLint Setup & Rules

This project uses ESLint with a comprehensive set of plugins and rules to ensure code quality, consistency, and maintainability. This guide covers the setup, configuration, and best practices for ESLint in this codebase.

Overview​

ESLint is configured with multiple plugins to enforce:

  • Code Quality: TypeScript, React, and React Native best practices
  • Consistency: Prettier formatting, import ordering
  • Performance: React hooks optimization, promise handling
  • Accessibility: React Native accessibility rules
  • Code Analysis: SonarJS code smell detection
  • Internationalization: i18next literal string checking
  • Documentation: Spell checking for comments and strings

Installation​

Install the required ESLint plugins and dependencies:

yarn add -D \
  eslint \
  @eslint/js \
  @typescript-eslint/parser \
  @typescript-eslint/eslint-plugin \
  eslint-plugin-react \
  eslint-plugin-react-hooks \
  eslint-plugin-react-native \
  eslint-plugin-prettier \
  eslint-config-prettier \
  eslint-plugin-import \
  eslint-plugin-promise \
  eslint-plugin-jest \
  eslint-plugin-spellcheck \
  eslint-plugin-i18next \
  eslint-plugin-sonarjs \
  globals

Configuration​

The ESLint configuration uses the flat config format and is defined in eslint.config.mjs:

import js from '@eslint/js';
import ts from '@typescript-eslint/eslint-plugin';
import parser from '@typescript-eslint/parser';
import i18next from 'eslint-plugin-i18next';
import importPlugin from 'eslint-plugin-import';
import jest from 'eslint-plugin-jest';
import prettier from 'eslint-plugin-prettier';
import promise from 'eslint-plugin-promise';
import react from 'eslint-plugin-react';
import reactHooks from 'eslint-plugin-react-hooks';
import reactNative from 'eslint-plugin-react-native';
import sonarjs from 'eslint-plugin-sonarjs';
import spellcheck from 'eslint-plugin-spellcheck';
import globals from 'globals';

export default [
  // -------------------------
  // Base JS recommended rules
  // -------------------------
  js.configs.recommended,

  // -------------------------
  // React Recommended
  // -------------------------
  {
    plugins: {
      react,
      'react-hooks': reactHooks,
      prettier,
      import: importPlugin,

      ecmaVersion: 'latest',

        ...globals.browser,

      },

      react: {
        createClass: 'createReactClass',
        pragma: 'React',

      },
      'import/resolver': {
        typescript: {},
      },
    },

    files: ['**/*.{js,jsx,ts,tsx}'],

    rules: {
      // -----------------------
      // Prettier
      // -----------------------
      'prettier/prettier': [
        'warn',

          bracketSpacing: true,

      // -----------------------

        },

      'promise/no-return-wrap': 'error',

Rule Categories​

Prettier​

Prettier is integrated with ESLint to enforce consistent code formatting:

  • Single quotes: Enforces single quotes for strings
  • Semicolons: Requires semicolons at the end of statements
  • Trailing commas: ES5-compatible trailing commas in objects and arrays
  • Print width: Maximum line length of 100 characters
  • Tab width: 2 spaces for indentation

Best Practice: Run eslint --fix to automatically format code according to Prettier rules.

React & React Hooks​

Rules enforce React best practices:

  • react-hooks/rules-of-hooks: Ensures hooks are called in the correct order
  • react/jsx-no-bind: Prevents inline function creation in JSX (forces useCallback)
  • react/function-component-definition: Enforces arrow function components
  • react/self-closing-comp: Requires self-closing tags where appropriate
  • react/jsx-pascal-case: Enforces PascalCase for component names

Best Practice: Use useCallback and useMemo to optimize performance instead of inline functions.

TypeScript​

TypeScript-specific rules ensure type safety:

  • @typescript-eslint/no-unused-vars: Catches unused variables and parameters
  • @typescript-eslint/no-explicit-any: Prevents use of any type (encourages proper typing)

Best Practice: Always provide explicit types instead of using any. Use unknown when the type is truly unknown.

React Native​

React Native specific rules:

  • Ensures proper React Native patterns
  • Prevents common React Native anti-patterns

SonarJS​

Code quality rules that detect code smells:

  • Cognitive Complexity: Limits function complexity
  • Identical Expressions/Functions: Prevents code duplication
  • Collapsible If: Simplifies nested conditions
  • Prefer Object Literal: Encourages object literal syntax
  • Prefer Single Boolean Return: Simplifies boolean returns

Best Practice: Keep functions simple and focused. Extract complex logic into smaller, well-named functions.

Promise​

Ensures proper promise handling:

  • promise/always-return: Promises must return values
  • promise/catch-or-return: Promises must have error handling
  • promise/param-names: Enforces consistent promise parameter naming

Best Practice: Always handle promise errors with .catch() or try/catch in async functions.

Import Ordering​

Enforces consistent import ordering:

  1. Built-in Node.js modules
  2. External packages
  3. Internal modules
  4. Parent/sibling/index imports
  5. Object imports
  6. Type imports

Best Practice: Let ESLint auto-fix import order with --fix flag. Keep imports grouped and alphabetized.

Spellcheck​

Validates spelling in:

  • Comments
  • Strings
  • Identifiers
  • Template literals

Best Practice: Add project-specific terms to the skipWords array in the configuration.

i18next​

Enforces internationalization best practices:

  • Warns about literal strings that should be translated
  • Ignores attributes like testID, accessibilityLabel, etc.

Best Practice: Use translation functions for user-facing strings, keep technical identifiers as literals.

Best Practices​

1. Use Arrow Function Components​

All components should be arrow functions:

// ✅ Good
const MyComponent: React.FC<Props> = ({ prop1, prop2 }) => {
  return <View />;
};

// ❌ Bad
function MyComponent({ prop1, prop2 }: Props) {
  return <View />;
}

2. Avoid Inline Functions in JSX​

Use useCallback for event handlers:

// ✅ Good
const handlePress = useCallback(() => {
  onPress(index);
}, [index, onPress]);

return <TouchableOpacity onPress={handlePress} />;

// ❌ Bad
return <TouchableOpacity onPress={() => onPress(index)} />;

3. Use TypeScript Types​

Avoid any, use explicit types:

// ✅ Good
const handleChange = (value: string): void => {
  setValue(value);
};

// ❌ Bad
const handleChange = (value: any) => {
  setValue(value);
};

4. Handle Promises Properly​

Always handle promise errors:

// ✅ Good
try {
  const result = await fetchData();
  return result;
} catch (error) {
  console.error(error);
  throw error;
}

// ❌ Bad
const result = await fetchData();
return result;

5. Organize Imports​

Let ESLint auto-fix import order:

// ✅ Good - Auto-sorted by ESLint
import React from 'react';
import { View, Text } from 'react-native';

import { Button } from '../../atoms/Button';
import { useTheme } from '../../theme';

// ❌ Bad - Manual ordering
import { useTheme } from '../../theme';
import React from 'react';
import { Button } from '../../atoms/Button';
import { View, Text } from 'react-native';

6. Keep Functions Simple​

Break down complex functions:

// ✅ Good
const calculateTotal = (items: Item[]): number => {
  return items.reduce((sum, item) => sum + item.price, 0);
};

// ❌ Bad - Too complex
const processOrder = (order: Order) => {
  // 50+ lines of nested logic
  if (order.items) {
    if (order.items.length > 0) {
      if (order.status === 'pending') {
        // ... many more nested conditions
      }
    }
  }
};

7. Use Prettier Configuration​

Ensure your editor is configured to format on save using the Prettier configuration from ESLint.

8. Run ESLint Before Committing​

Use Git hooks (like Lefthook) to run ESLint before commits:

# lefthook.yml
pre-commit:
  parallel: true
  commands:
    lint:
      glob: "*.{js,ts,jsx,tsx}"
      run: npx eslint --fix

Common Issues & Solutions​

Issue: "Inline function in JSX"​

Solution: Wrap the function in useCallback:

const handlePress = useCallback(() => {
  onPress(index);
}, [index, onPress]);

<TouchableOpacity onPress={handlePress} />

Issue: "Unused variable"​

Solution: Remove the variable or prefix with _ to indicate intentional unused:

const { _unused, used } = data; // _ prefix indicates intentional

Issue: "Promise must be handled"​

Solution: Add error handling:

promise
  .then(result => {
    // handle result
  })
  .catch(error => {
    // handle error
  });

Issue: "Import order"​

Solution: Run eslint --fix to auto-fix import order.

Issue: "Spellcheck error"​

Solution: Add the word to skipWords array in ESLint config if it's project-specific terminology.

Running ESLint​

Check for Issues​

yarn eslint .

Auto-fix Issues​

yarn eslint . --fix

Check Specific Files​

yarn eslint src/components/Button/*.tsx

Check with TypeScript​

yarn tsc --noEmit && yarn eslint .

IDE Integration​

VS Code​

Install the ESLint extension and configure in .vscode/settings.json:

{
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    "typescript",
    "typescriptreact"
  ]
}

WebStorm / IntelliJ​

  1. Go to Settings → Languages & Frameworks → JavaScript → Code Quality Tools → ESLint
  2. Enable ESLint
  3. Enable "Run eslint --fix on save"
