Header

A navigation header component with back button, title, and optional left/right components.

Category: Molecule
Dependencies: Icon, Text, @react-navigation/native (for navigation functionality)

Interactive Preview​

Edit the code to customize the Header:

Usage​

import { Header } from '@zaisustech/atomic-ui';

function MyScreen() {
  return (
    <Header
      title="My Screen"
      showBackButton={true}
      onBackPress={() => {
        // Handle back navigation
        navigation.goBack();
      }}
    />
  );
}

Props​

title​

Type: string

Optional title text displayed in the center of the header. If the title is too long, it will wrap to multiple lines (max 2 lines).

showBackButton​

Type: boolean

Default value: true

Whether to show the back button on the left side. When true, displays a circular icon button with chevron-back icon. When false, shows a placeholder or leftComponent if provided.

onBackPress​

Type: () => void

Optional callback fired when the back button is pressed. If not provided, uses React Navigation's navigation.goBack() automatically.

leftComponent​

Type: React.ReactNode

Optional custom component to display on the left side. Only used when showBackButton is false. If not provided and showBackButton is false, displays a placeholder to maintain layout.

rightComponent​

Type: React.ReactNode

Optional custom component to display on the right side. Can be used to add action buttons, icons, or other controls.

style​

Type: StyleProp<ViewStyle>

Optional custom styles for the header container.

titleStyle​

Type: StyleProp<TextStyle>

Optional custom styles for the title text.

Examples​

Basic Usage​

import { Header } from '@zaisustech/atomic-ui';

<Header title="My Screen" showBackButton={true} />

Without Back Button​

<Header title="Home Screen" showBackButton={false} />

With Custom Back Handler​

<Header
  title="Custom Back"
  showBackButton={true}
  onBackPress={() => {
    // Custom navigation logic
    console.log('Custom back handler');
    navigation.goBack();
  }}
/>

With Right Component​

import { Header, Icon } from '@zaisustech/atomic-ui';

<Header
  title="Settings"
  showBackButton={true}
  rightComponent={
    <Icon
      name="settings"
      type="ion"
      size={24}
      variant="circle"
      onPress={() => console.log('Settings pressed')}
    />
  }
/>

With Custom Left Component​

<Header
  title="Menu"
  showBackButton={false}
  leftComponent={
    <Icon
      name="menu"
      type="ion"
      size={24}
      variant="circle"
      onPress={() => console.log('Menu pressed')}
    />
  }
/>

With Both Components​

<Header
  title="Profile"
  showBackButton={true}
  leftComponent={
    <Icon
      name="menu"
      type="ion"
      size={24}
      variant="circle"
      onPress={() => console.log('Menu pressed')}
    />
  }
  rightComponent={
    <Icon
      name="settings"
      type="ion"
      size={24}
      variant="circle"
      onPress={() => console.log('Settings pressed')}
    />
  }
/>

Long Title​

The header automatically handles long titles by wrapping to multiple lines (max 2 lines):

<Header
  title="This is a very long header title that will wrap to multiple lines"
  showBackButton={true}
/>

Custom Styling​

import { StyleSheet } from 'react-native';

const customHeaderStyle = StyleSheet.create({
  customHeader: {
    backgroundColor: '#FF6B6B',
  },
  customTitle: {
    color: '#FFFFFF',
    fontSize: 20,
  },
});

<Header
  title="Custom Styled"
  style={customHeaderStyle.customHeader}
  titleStyle={customHeaderStyle.customTitle}
/>

Notes​

  • React Navigation Required: The Header component uses React Navigation's useNavigation hook, so it must be used within a NavigationContainer from @react-navigation/native in your app.

  • Back Button Behavior: If onBackPress is not provided, the header automatically uses navigation.goBack() when the back button is pressed.

  • Safe Area: The header automatically handles safe area insets for the status bar on both iOS and Android.

  • Layout: The header uses a flex layout with three sections: left (back button or placeholder), center (title), and right (optional component or placeholder).

  • Accessibility: The back button includes proper accessibility labels for screen readers.
