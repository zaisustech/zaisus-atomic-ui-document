File System API Reference

Reference for @zaisustech/file-system aligned with libs/file-system. Exports include context, widgets, hooks, components, types, and utils.

Context & provider​

FileManagerProvider​

Wrap your app or screen tree when using FileManagerContainer. Provides view mode (list/grid), sort key/direction, and wraps children with GestureHandlerRootView and KeyboardProvider.

PropTypeDefaultDescription
childrenReact.ReactNode—Required.
defaultViewModeFileViewMode'list'Initial view: 'list' or 'grid'.
defaultSortKeyFileSortKey'name'Initial sort key.
defaultSortDirectionFileSortDirection'asc'Initial sort direction.

Hooks (context)​

  • useFileManagerContext — view mode, sort state (when inside provider).
  • useRequiredFileManagerContext — same, but throws if used outside provider.

Widgets​

FileManagerContainer​

Full file manager UI: toolbar (search, sort, view mode, create folder, or selection toolbar), list or grid, modals. Must be used inside FileManagerProvider.

PropTypeDescription
pathstringRequired. Directory URI to display.
onSelect(file: FileItem) => voidOptional. Called when user taps a file or folder.
renderItem(args: { item: FileItem; viewMode: FileViewMode }) => React.ReactElement | nullOptional. Custom row/cell renderer.

FileDestinationModalWidget​

Reusable destination picker for copy/move (used internally; can be used in custom flows).

Hooks​

useFileManager​

Read directory contents and manage file list state.

Arguments: { path: string }

Returns:

PropertyTypeDescription
filesFileItem[]Directory contents.
isLoadingbooleanLoading state.
errorError | nullError if any.
titlestringLast path segment (e.g. "Documents").
refresh() => voidReload directory.

useFileActions​

File and directory operations. Pass onRefresh so the list reloads after actions.

Arguments: { onRefresh?: () => void }

Returns: copy, move, remove, rename, duplicate, createDirectory

  • copy(item, destDir, newName?)
  • move(item, destDir)
  • remove(item)
  • rename(item, newName)
  • duplicate(item) — copies into same folder with (n) suffix
  • createDirectory(parentPath, name)

useFileSort​

Sort file items by name, date, size, or kind. Can be controlled (pass sortKey / sortDirection / setters) or uncontrolled (initialKey / initialDirection).

Arguments: files, optional sortKey / sortDirection / setSortKey / setSortDirection, optional initialKey (default 'name'), initialDirection (default 'asc')

Returns: sortedFiles, sortKey, setSortKey, sortDirection, setSortDirection

useAppForegroundRefresh​

Calls a callback when the app comes to foreground.

Arguments: { onRefresh: () => void }

Components (custom UIs)​

ComponentDescription
FileListView / FileGridViewList and grid with data, keyExtractor, renderItem, onRefresh.
FileRowItem / FileGridItemContent-only display for one item (name, icon, meta).
FileRowItemWrapper / FileGridItemWrapperWrap item; handle press, long-press, selection, swipe (list).
FileToolBarSearch, create folder, sort, list/grid toggle.
FileSelectionToolBarSelect all, deselect all, delete, copy, move, cancel.
SearchToolBarText input + cancel.
ManageFolderModalCreate folder or rename.
SortBottomSheetModalSort by name/date/size/kind, asc/desc.
FileActionModalSingle-item: Copy, Move, Delete, Rename, Duplicate.
DeleteConfirmModalConfirm delete (single or multiple).

Types​

FileItem​

type FileItem = {
  uri: string;
  name: string;
  size: number;
  isDirectory: boolean;
  modifiedAt: number;
  fileType: FileType;
};

FileType​

type FileType = 'directory' | 'pdf' | 'video' | 'image' | 'audio' | 'document' | 'other';

FileViewMode​

type FileViewMode = 'list' | 'grid';

FileSortKey / FileSortDirection​

type FileSortKey = 'name' | 'date' | 'size' | 'kind';
type FileSortDirection = 'asc' | 'desc';

FileGroupListRow / FileGroupByType​

For groupFileItemsBy() (FlatList section-style data):

type FileGroupByType = 'name' | 'modifyDate';

type FileGroupListRow =
  | { type: 'header'; key: string; title: string }
  | { type: 'item'; key: string; item: FileItem };

Other types: FileSortState, FileIconDescriptor.

Utils​

FunctionDescription
formatSize(bytes)e.g. "1.0 KB"
formatModifiedAt(timestamp)e.g. "Jan 30, 2025"
getFileIcon(fileType)Returns FileIconDescriptor (e.g. { type: 'ion', name: 'folder' }).
getFileTypeFromName(name, isDirectory)Derive FileType from filename.
groupFileItemsBy(files, by)by: 'name' or 'modifyDate' → FileGroupListRow[] for section headers + items.

Example: hooks only (custom list)​

import { useFileManager, useFileActions, useFileSort } from '@zaisustech/file-system';
import * as FileSystem from 'expo-file-system';

const path = FileSystem.Paths.document.uri ?? '';

const { files, isLoading, error, title, refresh } = useFileManager({ path });
const { sortedFiles, sortKey, setSortKey, sortDirection, setSortDirection } = useFileSort({
  files,
  initialKey: 'name',
  initialDirection: 'asc',
});
const { copy, move, remove, rename, duplicate, createDirectory } = useFileActions({ onRefresh: refresh });

For full details and more examples, see libs/file-system/README.md.
