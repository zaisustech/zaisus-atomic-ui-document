# Atomic Design Pattern

Atomic Design is a methodology for creating design systems. It breaks down interfaces into fundamental building blocks that work together to create more complex UI components.

## Overview

The Atomic Design methodology consists of five distinct levels:

1. **Atoms** - The basic building blocks
2. **Molecules** - Simple combinations of atoms
3. **Organisms** - Complex components made of molecules and atoms
4. **Templates** - Page-level layouts
5. **Pages** - Specific instances of templates

This library implements the first three levels: **Atoms**, **Molecules**, and **Tokens** (foundation layer).

## Atoms

**Atoms** are the smallest, most basic components. They cannot be broken down further without losing their meaning. They serve as the foundational building blocks of your interface.

### Characteristics

- **Single responsibility**: Each atom has one clear purpose
- **Highly reusable**: Can be used across the entire application
- **No dependencies**: Atoms don't depend on other components (except tokens)
- **Minimal state**: Usually stateless or with very simple state

### Atoms in This Library

| Component | Purpose | Dependencies |
|-----------|---------|--------------|
| `Badge` | Display labels, counts, or status | Tokens (colors, spacing, radius) |
| `Button` | Interactive button element | `Text`, Tokens |
| `Divider` | Visual separator | Tokens (colors, spacing) |
| `Icon` | Icon display | Tokens (colors) |
| `ProgressBar` | Horizontal progress indicator | Tokens (colors, spacing) |
| `ProgressCircle` | Circular progress indicator | Tokens (colors) |
| `Spacer` | Flexible spacing element | Tokens (spacing) |
| `Text` | Typography component | Tokens (typography, colors) |
| `TextInputField` | Text input with label | `FormFieldContainer`, `Text`, Tokens |
| `Visible` | Conditional rendering wrapper | None |

### Example: Atom Usage

```tsx
import { Button, Text, Icon } from '@zaisustech/atomic-ui';

// Simple atom usage
<Text variant="Paragraph.default.regular">Hello World</Text>
<Button title="Click me" onPress={() => {}} />
<Icon name="heart" type="fa" size={24} />
```

## Molecules

**Molecules** are relatively simple combinations of atoms functioning together as a unit. They form simple, reusable UI components.

### Characteristics

- **Composed of atoms**: Built using one or more atoms
- **Single purpose**: Still focused on a specific function
- **Reusable**: Can be used in multiple contexts
- **May have state**: Can manage their own internal state

### Molecules in This Library

| Component | Purpose | Composed Of |
|-----------|---------|-------------|
| `Accordion` | Collapsible content section | `Text`, `Icon`, `Button` |
| `ActionSheet` | Modal action menu | `Button`, `Text`, `Icon` |
| `Card` | Container with shadow | `Text`, Tokens (shadows) |
| `Carousel` | Horizontal scrolling pages | `View`, `ScrollView` |
| `DropdownField` | Select dropdown | `TextInputField`, `FormFieldContainer`, `Icon`, `Text` |
| `EmptyState` | Empty state display | `Text`, `Icon`, `Button` |
| `FormFieldContainer` | Form field wrapper | `Text`, `Icon` |
| `ListItem` | List row item | `Text`, `Icon` |
| `Pager` | Page navigation | `Visible`, `FlatList` |
| `RichText` | Formatted text | `Text` |
| `SegmentedControl` | Segmented button control | `Text`, `Button` |
| `Snackbar` | Temporary notification | `Text`, `Icon` |
| `Stepper` | Number stepper | `Button`, `Text` |
| `TabView` | Tab navigation | `Text`, `View` |
| `Toast` | Toast notification | `Text`, `Icon`, `Button` |

### Example: Molecule Usage

```tsx
import { Card, ListItem, DropdownField } from '@zaisustech/atomic-ui';

// Molecules combine atoms
<Card variant="medium">
  <Text>Card content</Text>
</Card>

<ListItem
  title="Settings"
  subtitle="Manage preferences"
  leftElement={<Icon name="settings" />}
  rightElement={<Icon name="chevron-right" />}
/>
```

## Tokens (Foundation Layer)

**Tokens** are the foundational design decisions. They represent the smallest pieces of your design language and ensure consistency across components.

### Token Categories

1. **Colors**: Semantic and primitive color palettes
2. **Spacing**: Consistent spacing scale
3. **Typography**: Text styles and font definitions
4. **Shadows**: Elevation and shadow definitions
5. **Radius**: Border radius values

### How Tokens Support Components

All components consume tokens through the `useTheme()` hook:

```tsx
import { useTheme } from '@zaisustech/atomic-ui';

function MyComponent() {
  const { colors, spacing, typography, radius, shadows } = useTheme();
  
  // Use tokens for styling
  return (
    <View style={{
      backgroundColor: colors.primary,
      padding: spacing.md,
      borderRadius: radius.md,
    }}>
      <Text style={typography.header.large.bold}>
        Title
      </Text>
    </View>
  );
}
```

## Benefits of Atomic Design

### 1. **Consistency**
Components are built from the same atoms, ensuring visual and functional consistency across your application.

### 2. **Reusability**
Atoms and molecules can be reused in multiple contexts, reducing code duplication.

### 3. **Maintainability**
Changes to atoms propagate through molecules, making updates easier and more predictable.

### 4. **Scalability**
The hierarchical structure makes it easy to add new components without affecting existing ones.

### 5. **Collaboration**
Clear component boundaries make it easier for teams to work together.

### 6. **Testing**
Small, focused components are easier to test in isolation.

## Best Practices

### 1. **Start with Atoms**
When building new features, start by checking if existing atoms can be used or combined.

### 2. **Compose, Don't Duplicate**
Instead of creating similar components, compose molecules from existing atoms.

### 3. **Use Tokens**
Always use design tokens instead of hardcoded values for colors, spacing, etc.

### 4. **Keep Atoms Simple**
Atoms should be simple and focused. If an atom becomes complex, consider splitting it.

### 5. **Molecules Should Be Meaningful**
Molecules should represent a meaningful UI pattern, not just a random combination of atoms.

### 6. **Document Dependencies**
When creating molecules, document which atoms they depend on.

## Example: Building a Component

Let's see how atomic design works in practice by building a `SearchBar` molecule:

```tsx
// Step 1: Use existing atoms
import { TextInputField, Icon, Button } from '@zaisustech/atomic-ui';

// Step 2: Compose them into a molecule
function SearchBar({ onSearch, placeholder }) {
  const [query, setQuery] = useState('');
  
  return (
    <View style={{ flexDirection: 'row', gap: 8 }}>
      <TextInputField
        placeholder={placeholder}
        value={query}
        onChangeText={setQuery}
        leftElement={<Icon name="search" type="fa" />}
      />
      <Button
        title="Search"
        onPress={() => onSearch(query)}
      />
    </View>
  );
}
```

This `SearchBar` molecule is composed of:
- `TextInputField` (atom)
- `Icon` (atom)
- `Button` (atom)

## Summary

The Atomic Design pattern provides a systematic approach to building UI components:

- **Tokens** provide the foundation with design values
- **Atoms** are the basic building blocks
- **Molecules** combine atoms into functional components
- **Theme** system ties everything together

This structure ensures consistency, reusability, and maintainability across your entire application.
